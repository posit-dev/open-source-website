<h3><a href="/people/simon-couch/">Simon Couch</a>: thinking</h3>

<p><a data-t="39">Simon what's something that Posit Assistant does really well, and how did you build that?</a> <a data-t="44">I wanted to talk about something that is a little bit under the hood, but I think is</a> <a data-t="48">actually really important to making Posit Assistant feel as good as it does.</a> <a data-t="53">And that thing is thinking, otherwise known as reasoning, which lets the models think</a> <a data-t="58">out loud to themselves before they take actions or before they respond to you.</a></p>

<p><a data-t="63">So as an example, I have this Posit Assistant conversation pulled up, and I've asked the</a> <a data-t="68">agent to make a couple plots of this forested data set.</a> <a data-t="76">And it's gone on and written some ggplot code.</a> <a data-t="81">Because Cloud Sonnet supports thinking, there's this little light bulb icon, and when I click</a> <a data-t="86">it I can change to a number of thinking settings, but I'll just leave it on high for the purposes</a> <a data-t="91">of this.</a></p>

<p><a data-t="97">So let's suppose instead of the default ggplot blue and red, I wanted to change to a nice</a> <a data-t="103">subtle green and yellow.</a> <a data-t="106">If I go ahead and submit that follow-up, we'll see in the UI that the model is thinking out</a> <a data-t="111">loud to itself, and once it's done thinking, that little thinking block goes away and the</a> <a data-t="116">model starts taking action.</a></p>

<p><a data-t="109">So one of the things that we saw coming when we implemented thinking was that the models</a> <a data-t="113">might take more reasonable actions or write better code, but one of the things that surprised</a> <a data-t="118">us is that the models also seem to communicate with you more clearly and be more respectful</a> <a data-t="123">of your attention when they have that chance to think out loud to themselves first.</a> <a data-t="128">So we see in this example that in its reply to us, it's just given a one-sentence summary</a> <a data-t="134">of its actions because that's really all that's needed in order to conclude that follow-up.</a></p>

<blockquote><a data-t="118">the models also seem to communicate with you more clearly and be more respectful</a> <a data-t="123">of your attention when they have that chance to think out loud to themselves first.</a></blockquote>

<h3><a href="/people/sara-altman/">Sara Altman</a>: predictive modeling</h3>

<p><a data-t="210">Sarah, what's something LLM struggle with when it comes to data analysis, and how did</a> <a data-t="214">you build around that for Posit Assistant?</a> <a data-t="217">When working on Posit Assistant, we noticed that it would often make methodological mistakes</a> <a data-t="221">when building predictive models, like training on the test set or switching metrics mid-process.</a> <a data-t="227">We did two primary things to address this problem.</a></p>

<p><a data-t="230">The first is that we fixed the prompting to generally encourage best practices and discourage</a> <a data-t="234">bad ones, and the second is that we created a predictive modeling skill.</a> <a data-t="239">Skills are additional pieces of prompting and code that are invoked only when needed.</a> <a data-t="244">In Posit Assistant, you can invoke the predictive modeling skill either by referring to it by</a> <a data-t="248">name or by mentioning that you're working on a predictive model.</a></p>

<p><a data-t="252">The skill is an attempt to get the entire process of modeling right.</a> <a data-t="256">So in R, it defaults to tidy models functions, which have a consistent interface and sensible</a> <a data-t="260">defaults, making it more likely that the model will make good decisions.</a> <a data-t="264">It also instructs Posit Assistant to follow sound modeling practices, splitting your data</a> <a data-t="269">and holding out the test set, resampling, starting with baseline model, iterating one</a> <a data-t="273">change at a time, and comparing models in the same metric set.</a></p>

<p><a data-t="278">After each model, it shows you a running comparison table so that you can track progress, and</a> <a data-t="282">it also asks for your permission before ever touching the test set.</a> <a data-t="286">This is because once you use the test set to make decisions, you've lost your unbiased</a> <a data-t="289">estimate of how the model will perform on unseen data.</a> <a data-t="294">With this skill, it's more likely that you and Posit Assistant together will make good</a> <a data-t="297">predictive modeling choices and avoid mistakes that will interfere with or bias your results.</a></p>
