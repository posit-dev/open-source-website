<p><a data-t="0">Two AI products. They're built on top of the same base model, so they must be the same, right?</a> <a data-t="5">Well, not exactly. AI products may be built on the same base model, but the product developers</a> <a data-t="11">decide what instructions they get and what tools they have access to, and tools are where the magic</a> <a data-t="17">happens.</a></p>

<p><a data-t="17">Because at its core, an LLM alone is just text in and text out.</a> <a data-t="23">It can write you Python code,</a> <a data-t="23">but by itself it doesn't have any way to run that Python code. It's just text.</a> <a data-t="29">A tool is sort</a> <a data-t="29">of like a predefined function that the model is allowed to request, and using the tool, the LLM</a> <a data-t="35">might say something like, hey, I want to run this Python code, and then an application like </a><a href="/software/positron/">Positron</a> <a data-t="41">goes and runs it and hands the results back to the LLM.</a></p>

<blockquote><a data-t="17">Because at its core, an LLM alone is just text in and text out.</a> <a data-t="23">It can write you Python code,</a> <a data-t="23">but by itself it doesn't have any way to run that Python code. It's just text.</a></blockquote>

<p><a data-t="47">Again, the LLM is just taking text in</a> <a data-t="47">and outputting more text.</a> <a data-t="52">So when one AI product can, say, browse the internet for you and another</a> <a data-t="52">one can't, even though it has the same access to the same underlying model, it's because they have</a> <a data-t="57">different tools.</a></p>
