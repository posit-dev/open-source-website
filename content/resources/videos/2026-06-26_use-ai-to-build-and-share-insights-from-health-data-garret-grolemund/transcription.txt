<p><a data-t="5926">This architecture has some real benefits.</a> <a data-t="5929">One, it's quite reliable.</a> <a data-t="5931">The database is running SQL.</a> <a data-t="5932">The LLM's writing SQL.</a> <a data-t="5934">If anything hallucinates, it just tries again</a> <a data-t="5936">before it messes anything up.</a> <a data-t="5939">It works pretty well.</a> <a data-t="5940">I mean, it's well-tested at this point.</a> <a data-t="5942">Two, it's safe for a number of reasons.</a> <a data-t="5944">One, querychat won't let the LLM run</a> <a data-t="5948">or it won't let the database run destructive queries.</a> <a data-t="5951">But even if it did, that database was spun up</a> <a data-t="5954">at the time the Shiny app was launched.</a> <a data-t="5957">The next time someone launches the Shiny app,</a> <a data-t="5959">a new database will be spun up.</a> <a data-t="5960">Like, there's nothing permanent that could be destroyed here.</a> <a data-t="5963">Third, it's reproducible.</a> <a data-t="5965">You can give, your user will have access to that SQL code.</a> <a data-t="5968">You can even feature it inside your Shiny app.</a> <a data-t="5971">So if they wanted to,</a> <a data-t="5972">they could then collect it and rerun it, use it later.</a></p>

<p><a data-t="5976">And then four, it's kind of private</a> <a data-t="5978">in that the LLM doesn't ever really see the raw data,</a> <a data-t="5982">but it can query the data</a> <a data-t="5984">to give the user responses in the chat.</a> <a data-t="5987">And that can certainly reveal things about the data.</a> <a data-t="5989">So it's definitely not, like, secure in and of itself.</a> <a data-t="5992">This is a situation, a lot like Positive Assistant itself,</a> <a data-t="5995">where you would want to trust the model that you are using.</a> <a data-t="6002">And what we've discovered working with clients,</a> <a data-t="6002">working with AI for about a year now,</a> <a data-t="6004">is there's no way around trusting the model.</a> <a data-t="6007">If you don't trust your model, don't use it with your data.</a> <a data-t="6009">And it is very easy these days</a> <a data-t="6012">to secure a use of a model that you can trust.</a> <a data-t="6017">The model providers want to make the models trustworthy</a> <a data-t="6020">and secure, because they know</a> <a data-t="6020">no one would use them otherwise.</a> <a data-t="6022">So normally people just arrange with Anthropic, whatever,</a> <a data-t="6026">to have their data not be used.</a></p>
